# cloud

## 职责

当前不是一个可用源码模块，而是一个“历史残留说明”模块：当前 `easycontrol/settings.gradle` 仅声明 `:app` 与 `:server`，`:cloud` 已不在当前构建中；它更多用于解释历史商业版/私有能力差异。

## 接口定义（可选）

> 当前仓库未提供 `easycontrol/cloud/` 源码，且当前开源构建也不再声明该模块，因此没有可记录的实际 API。

### 已知事实
| 项 | 事实 |
|----|------|
| 当前 Gradle 声明 | 已不再包含 `include ':cloud'`，仅保留 `:app` 与 `:server` |
| 目录现状 | 未检测到 `easycontrol/cloud/` |
| 相关文档 | 当前公开文档已将激活/历史发行版差异描述为说明信息，而非当前构建前提 |

## 行为规范

### 历史差异排查
**条件**: 需要分析开源版与官方/商业版差异。
**行为**: 将 cloud/激活相关能力视为仓库外部或历史残留，不要默认其源码仍存在于当前工作区。
**结果**: 在排查 issue、迁移旧补丁或阅读旧文档时保持边界清晰。

### 构建前检查
**条件**: 需要执行全量 Gradle 构建。
**行为**: 以当前 `settings.gradle` 和实际目录结构为准，不再把 `:cloud` 当作当前构建依赖；若历史文档与代码冲突，以代码为准。
**结果**: 避免基于旧文档错误判断当前构建结构。
